> [!quote] 網頁安全是一門關於搶銀行的學問

網頁安全的核心在於如何防止惡意使用者假冒受害者去操作任何敏感服務（像是銀行帳號存取）。舉例來說，有些網站會讓你存取別的網站，並且用從你那裡拿網路餅乾（Cookie）做些機密操作。

網頁安全也有幾個問題。瀏覽器（User Agent）很強大，但也過於「熱心」。當它發送請求時，常常自動附上你的身份憑證。伺服器與客戶端兩端也都在無條件的信任不可信的輸入。而漏洞往往就在這些被錯誤信任與錯誤解析的訊息中。

也就是說，網頁安全其實重點在於如何管好本地瀏覽器資料、如何驗證請求來源。

# 搶銀行
假設攻擊者有個網站，而他在網站內的 HTML 中插入一個會向銀行發送請求的標籤，像是：
```html
<img src="http://bank.com/transfer?to=Red&amount=1000000">
```
當受害者的瀏覽器解析到這個 `<img>` 標籤時，它就會自動向 `bank.com` 發出 GET 請求。而若受害者已經登入過銀行，瀏覽器就會自動附上 `bank.com` 的網路餅乾！於是錢就順理成章的 `transfer to Red` 了。

# 動手觀察

`F12` -> Network -> 隨便一個請求，看看右邊 Request Headers，那個 Cookie 就是瀏覽器只要訪問網站，只要有網路餅乾就會自動把它塞進封包的證明。
